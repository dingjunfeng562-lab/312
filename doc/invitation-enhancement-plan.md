# 邀请码功能扩展方案

## 一、功能需求

### 1. 使用记录详情
- 查看邀请码的使用者用户名
- 记录使用时间
- 记录使用者 IP 地址
- 在前端提供"查看详情"按钮

### 2. 过期时间管理
- 创建邀请码时可设置有效期（7天/30天/90天/365天/永久）
- 系统自动检测过期状态
- 过期的邀请码无法使用
- 前端显示过期状态（红色标注）

### 3. 超级管理员权限
- 在 `auth` 表增加 `admin_level` 字段
- baishuwan 和 root 设置为 `admin_level = 2`
- 高级功能（如生成带过期时间的邀请码）需要超级管理员权限
- 普通管理员只能查看和删除

## 二、数据库设计

### invitation 表扩展字段

| 字段名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| expires_at | DATETIME | NULL | 过期时间（NULL=永久有效） |
| creator_id | BIGINT | 0 | 创建者ID（0=系统） |
| creator_name | VARCHAR(100) | 'system' | 创建者用户名 |
| notes | VARCHAR(500) | '' | 备注信息 |
| used_at | DATETIME | NULL | 实际使用时间 |
| used_ip | VARCHAR(50) | '' | 使用者IP地址 |

### auth 表扩展字段

| 字段名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| admin_level | INT | 1 | 1=普通管理员, 2=超级管理员 |

## 三、API 接口设计

### 1. 高级邀请码生成
- **路由**：`POST /admin/invitation/generate-advanced`
- **权限**：超级管理员
- **请求体**：
```json
{
  "type": "vip",
  "quota": 100,
  "number": 5,
  "expires_days": 30,
  "notes": "测试批次"
}
```

### 2. 查询使用记录详情
- **路由**：`GET /admin/invitation/usage/:code`
- **权限**：管理员
- **响应体**：
```json
{
  "code": "vip-abc123...",
  "quota": 100,
  "type": "vip",
  "used": true,
  "created_at": "2026-07-08 10:00:00",
  "expires_at": "2026-08-07 10:00:00",
  "is_expired": false,
  "creator_name": "baishuwan",
  "used_by_user": "user123",
  "used_at": "2026-07-08 11:00:00",
  "used_ip": "192.168.1.100",
  "notes": "测试批次"
}
```

### 3. 查询过期邀请码列表
- **路由**：`GET /admin/invitation/check-expired`
- **权限**：管理员
- **响应体**：
```json
{
  "status": true,
  "data": [
    {
      "code": "vip-expired123",
      "expires_at": "2026-06-01 00:00:00",
      "creator_name": "baishuwan"
    }
  ]
}
```

## 四、前端组件设计

### 1. InvitationTable 表格扩展
- 新增列：创建者、过期时间、使用详情
- 过期邀请码红色标注
- "查看详情"按钮（仅已使用的邀请码显示）

### 2. InvitationUsageDialog 使用详情弹窗
- 显示使用者用户名
- 显示使用时间
- 显示使用者 IP
- 显示创建者和备注

### 3. InvitationAdvancedDialog 高级生成弹窗
- 过期时间下拉选择器（7/30/90/365 天/永久）
- 备注信息输入框
- 仅超级管理员可见

## 五、风险与缓解

### 1. 数据库迁移风险
- **风险**：生产环境数据丢失
- **缓解**：迁移前备份数据库

### 2. 权限系统变更风险
- **风险**：普通管理员无法访问必要功能
- **缓解**：仅对高级生成功能设限

### 3. 历史数据兼容性
- **风险**：历史邀请码缺少新字段
- **缓解**：数据库字段设置 DEFAULT 值

### 4. IP 获取准确性
- **风险**：代理/CDN 环境下获取代理 IP
- **缓解**：优先读取 X-Real-IP header
